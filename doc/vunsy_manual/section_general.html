Sections Idea

sections are the main pages of your website, it's the pages that everyone see,

how it works

database holds the pages names , ID, view permissions and so in a table called section , section could be nested like the directory hirarchy +index
|--+section1
|--+section2
|--|--+subsectionI
|--|--+subsectionII
|--|--|--+subsubsectionAlpha
|--|--+subsectionIII
|--section3
 vunsy loads the section that mentioned in the URL, it gots the ID then loads all the content related to that section and display it, for example if the URL like that www.your-site.com/index.php/3 then it loads the section corresponding to the ID=3 and display it if exists or display the index page if not exists

Details about it

'section' => array(
'parent_section' => array( 'type'=>'BIGINT','null'=>TRUE ),
'name' => array( 'type'=>'VARCHAR', 'constraint'=>100,'null'=>TRUE) ,
'sort' => array( 'type'=>'INT','null'=>TRUE ),
'view' => array( 'type'=>'LONGTEXT','null'=>TRUE)
) , that's the installation script that section has,
parent_section: the id of the parent section
name : section title that displays in the title tag
sort: the section order in the sections queue, it's useful in displying the sections lists, menus, tree and so
view: the view permission of the section, if valid the contents are shown, if not permission denied page is shown

building it

make sure you build your sections well they define the structure of your website
you can use the sections manager to add remove or edit your sections
